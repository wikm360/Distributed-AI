## How to Contribute

### 1. Fork and Clone

You may fork this repository **solely for the purpose of creating contributions** that will be submitted back to the original repository.

```bash
git clone https://github.com/wikm360/Distributed-AI.git
cd mediapipe
```

### 2. Create a Branch

Create a feature branch for your contribution:

```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes

- Write clean, well-documented code
- Follow the existing code style and conventions
- Add comments where necessary
- Update documentation if needed
- Test your changes thoroughly

### 4. Commit Your Changes

Write clear, descriptive commit messages:

```bash
git commit -m "Add: description of your changes"
```

### 5. Push and Create Pull Request

Push your branch and create a pull request:

```bash
git push origin feature/your-feature-name
```

Then create a pull request on the original repository with:
- A clear title and description
- Reference to any related issues
- Screenshots or examples if applicable

## Contribution Guidelines

### Code Style

- Follow Dart/Flutter style guidelines
- Use meaningful variable and function names
- Keep functions focused and single-purpose
- Add documentation comments for public APIs

### What Can Be Contributed

We welcome contributions in the following areas:

- **Bug Fixes**: Fixes for identified bugs and issues
- **Performance Improvements**: Optimizations that improve speed or efficiency
- **UI/UX Enhancements**: Improvements to user interface and experience
- **Documentation**: Improvements to code comments, README, or documentation
- **Code Quality**: Refactoring, code cleanup, and best practices
- **Feature Requests**: New features that align with the project's research goals

### What Cannot Be Contributed

- Standalone forks or independent projects based on this code
- Commercial use or monetization of the code
- Distribution of the code to third parties
- Removal of proprietary notices or copyright information

## Review Process

1. All pull requests will be reviewed by the project maintainer
2. The maintainer may request changes or modifications
3. The maintainer reserves the right to accept, reject, or modify any contribution
4. Accepted contributions will be merged into the main repository
